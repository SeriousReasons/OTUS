# Вход в Систему Без Пароля

### 1. Использование Режима Восстановления

**Шаги:**
- Перезагрузите и при загрузке выберите опцию восстановления (Recovery Mode) в меню GRUB.
- Выберите 'root' для доступа к корневой оболочке.

**Преимущества:**
- Простота использования.
- Не требует дополнительных инструментов.

**Недостатки:**
- Не всегда доступен в зависимости от настроек безопасности системы.

### 2. Изменение Параметров Загрузки в GRUB

Модификация параметров загрузки для получения root-доступа через bash.

**Шаги:**
- В меню GRUB нажмите `e` и добавьте `init=/bin/bash` к строке загрузки.
- Нажмите `Ctrl+X` для старта.

**Преимущества:**
- Позволяет получить немедленный доступ к корневой оболочке.

**Недостатки:**
- Требует временного изменения параметров загрузки.
- Может быть заблокировано настройками безопасности GRUB.

### 3. Использование Live CD/USB

Загрузка с внешнего носителя для доступа к файловой системе установленной системы.

**Шаги:**
- Загрузитесь с Live CD/USB.
- Используйте `chroot` для смены корня на файловую систему установленной системы.

**Преимущества:**
- Обеспечивает полный доступ к файловой системе.
- Не требует изменения системных файлов.

**Недостатки:**
- Требуется подготовка внешнего загрузочного устройства.

## Сравнение Методов

- **Доступность:** Режим восстановления и изменение параметров загрузки в GRUB зависят от настроек безопасности и могут быть недоступны в некоторых конфигурациях. Метод с Live CD/USB требует подготовки, но обычно всегда доступен.
- **Уровень доступа:** Все три метода предоставляют root-доступ, но метод с Live CD/USB может предоставить больше возможностей для работы с файловой системой.
- **Сложность:** Использование режима восстановления является самым простым, в то время как работа с Live CD/USB может потребовать дополнительных знаний и подготовки.

# Отчет по Настройке и Управлению LVM и initrd

Этот документ описывает процесс установки операционной системы с использованием LVM (Logical Volume Manager), переименование Volume Group (VG) после установки, а также добавление модуля в initrd.

# Переименование Volume Group

1. Проверка текущего имени VG:
   `vgs`
2. Переименование VG:
   vgrename старое_имя_группы_томов новое_имя_группы_томов
3. Обновление `/etc/fstab`, `/etc/default/grub`, и перегенерация initramfs:
   `update-initramfs -u`
4. Обновление GRUB:
   `update-grub`

# Добавление Модуля в initrd

1. Создание конфигурационного файла для dracut:
   `echo "add_drivers+=\" virtio_net \"" | sudo tee /etc/dracut.conf.d/virtio_net.conf`
2. Пересоздание initrd:
   `sudo dracut -f`
