# Вход в Систему Без Пароля

### 1. Использование Режима Восстановления (Не удалось реализовать на Debian 12)

- При загрузке в меню GRUB выбираю recovery mode.
- Затем выбираю 'root' для доступа к корневой оболочке.

**Приемущества:**
- Простота и удобство использования.
- Не требует внешних устройств или инструментов.

**Недостатки:**
- Возможные ограничения доступности из-за настроек безопасности системы.

### 2. Изменение Параметров Загрузки в GRUB (Скриншот 1-2)

- В меню GRUB нажимаю `e` и добавляю `init=/bin/bash` к строке загрузки, которая начинатеся c linux.
- Для запуска нажимаю `Ctrl+X`.

**Приемущества:**
- Быстрый доступ к корневой оболочке.

**Недостатки:**
- Необходимость внесения изменений в параметры загрузки.

### 3. Использование Live CD/USB (скриншот 3)ь.

- Загружаюсь с Live CD/USB.
- Применяю `chroot` для смены корня на файловую систему установленной системы.

**Преимущества:**
- Полный доступ к файловой системе.
- Возможность работы даже если другие методы недоступны.

**Недостатки:**
- Требует наличия подготовленного загрузочного устройства.

## Сравнение Методов

Режим восстановления и изменение параметров в GRUB могут быть ограничены настройками безопасности, в то время как метод с Live CD/USB всегда доступен при наличии подготовленного устройства.
Все методы предоставляют root-доступ, но Live CD/USB дает больше возможностей для работы с системой.
Режим восстановления является самым простым, но не всегда доступен по умолчанию, как показала практика с Debian 12.
